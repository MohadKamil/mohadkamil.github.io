Automating WPF deployment Take 2

Posted on Aug 26, 2019

In my previous post I’ve talked about how to automate the deployment of a WPF application, using a small app the uses some feature of Azure active directiry as a demo.

While the method I showcased gets us there, it fells short, the main concern is that it requires rebuilding the app again to get the deployment package, this defeat the mantra of build once deploy everywhere, and opens up a seem for changes between the application binaries you tested, and what gets deployed, what we want to acheive is to build our app, preferably as part of a CI build, store the needed dlls and binraries of the application produced by the build, and use those in a package step to create our deployment artifact.

Since we are using Clickonce, a great solution is to use the mage command line tool, this is the same tool visual studio uses behind the scenes when you run the build with Publish target, and here we will step over visual studio and run the tool outselfs.
